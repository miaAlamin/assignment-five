What is the Document Object Model (DOM)

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of an HTML or XML document as a tree of objects that can be manipulated using programming languages like JavaScript. The DOM allows you to access and modify the content, structure, and styles of a webpage while it is being viewed, creating dynamic and interactive websites.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where a single event listener is added to a parent element to manage events triggered by its child elements. Instead of attaching an event listener to every child, the listener is added to a common ancestor, which then listens for events from its descendants. This leverages event bubbling, where events propagate up through the DOM hierarchy from the target element to its ancestors. Why Event Delegation is Useful: Improved Performance: Instead of attaching event listeners to multiple child elements (e.g., in a list with 100 items), you only attach one event listener to the parent. This reduces memory usage and improves performance, especially in large or dynamic DOM structures. Handles Dynamically Added Elements: If new child elements are added to the DOM after the initial page load (e.g., via AJAX or JavaScript), event delegation still works without needing to attach new event listeners for those elements. This is because the parent is still listening for events that bubble up from its descendants. Simplified Code: Managing event listeners from a single parent node is easier and more maintainable than assigning individual listeners to many child nodes.

How do you manipulate on elements attributes and styles useing the DOM?

To manipulate an element's attributes, use methods like `setAttribute()`, `getAttribute()`, or `removeAttribute()`. For styles, modify the `style` property or use `classList` to add, remove, or toggle classes.
